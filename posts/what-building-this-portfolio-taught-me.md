When I started building this portfolio, I thought the biggest challenge would be learning to code.

It wasn't.

The hardest part was learning how to deal with things not working.

Almost every feature I added broke something else. A new animation would affect the layout. Fixing the mobile navigation would somehow create another issue on desktop. Sometimes I'd spend an hour chasing a bug only to discover I had missed a single bracket or forgotten to save a file.

As frustrating as that sounds, that's where the real learning happened.

## Every problem had an answer

One thing I realized early on is that almost every error has already happened to someone else.

Whether it was a Next.js error, a React warning, or something that simply refused to render correctly, there was usually a solution. Sometimes ChatGPT or Claude knew immediately. Other times I had to search documentation or compare different answers until something finally clicked.

Instead of seeing errors as failures, I slowly started treating them like clues.

## Small changes beat big changes

In the beginning, I'd ask AI to rewrite huge sections of my website.

That almost never ended well.

Eventually I learned to work in much smaller steps. Change one component. Test it. If it works, move on. If it breaks, I know exactly what caused it.

That simple habit probably saved me dozens of hours.

## Learning how to ask better questions

I used to give AI prompts like:

> "My website is broken."

Not surprisingly, the answers weren't very useful.

Later I learned to be much more specific.

Instead of saying something was broken, I'd explain exactly what I expected to happen, what actually happened, and include the error message if there was one.

The better my prompt became, the better the solution became.

In a way, I wasn't just learning web development. I was learning how to communicate technical problems clearly.

## Progress doesn't always look impressive

Some days I'd spend three hours working on the website.

The result?

A button moved five pixels.

A loading animation became smoother.

One bug disappeared.

From the outside, it looked like almost nothing had changed.

But every tiny improvement made the site feel a little more polished than it did the day before.

## Finishing matters more than perfection

There are still things on this portfolio I'd like to improve.

There are animations I could make smoother.

Some components could definitely be written better.

And there are probably plenty of lines of code that would make an experienced developer cringe.

But that's okay.

A finished project teaches you far more than a perfect project that never gets published.

## Looking back

A few weeks ago, building a portfolio like this felt impossible.

Now it's something I can continuously improve.

I'm still nowhere near calling myself a developer, but that's never really been the goal.

The goal was to build something real, learn along the way, and prove to myself that even with almost no coding background, I could create something I'm genuinely proud to put my name on.

And that's exactly what this project became.
